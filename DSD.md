### 3.7 점수화 알고리즘 모듈

#### 기능 설명
본 모듈은 발표 종료 후 생성된 SessionSummary 및 SlideLog 데이터를 기반으로 시선, 자세, 제스처, 시간 관리 4개 카테고리 점수를 0~100 범위로 정량화하고, 가중합 방식으로 최종 발표 점수를 산출한다.  
가중치는 비언어적 커뮤니케이션 관련 선행 연구를 기반으로 설정하였다.

#### 블록 다이어그램

[SessionSummary + SlideLog]
            │
            ▼
 ┌───────────────┬───────────────┬───────────────┬───────────────┐
 ▼               ▼               ▼               ▼
[Gaze Score] [Pose Score] [Gesture Score] [Time Score]
 └───────────────┴───────────────┴───────────────┴───────────────┘
                            │
                            ▼
                  [Weighted Sum]
                            │
                            ▼
                     [ScoreResult]

#### 카테고리 및 가중치

| 카테고리 | 근거 | 가중치 |
|----------|------|--------|
| 시선 (Gaze) | 시각적 요소는 호감도에 영향, 과도 시 전달 방해 가능 | 0.30 |
| 자세 (Pose) | 안정감 형성 요소 | 0.25 |
| 제스처 (Gesture) | 보조적 표현 요소 | 0.15 |
| 시간 (Time) | 성실성(준비성) 반영 핵심 요소 | 0.30 |

※ 가중치 합 = 1로 정규화

#### 알고리즘

1) 시선 점수

:contentReference[oaicite:0]{index=0}

2) 자세 점수

:contentReference[oaicite:1]{index=1}

3) 제스처 점수

:contentReference[oaicite:2]{index=2}

4) 시간 점수

:contentReference[oaicite:3]{index=3}

## 7. 테스트 계획

### 7.1 단위 테스트

| 모듈 | 테스트 항목 | 도구 | 합격 기준 |
|------|------------|------|----------|
| 제스처 인식 | 주먹/손바닥 인식 정확도 | 테스트 영상 | 90% 이상 |
| 시선 분석 | 정면 응시 판정 정확도 | 샘플 데이터 | ±5% 오차 |
| 자세 분석 | 어깨 기울기 계산 | 좌표 데이터 | ±3° 이하 |
| 점수화 알고리즘 | 경계값 (0~100) 유지 | Python 테스트 | 정상 범위 |
| JWT 인증 | 토큰 생성/검증 | pytest | 100% 성공 |
| PDF 생성 | 파일 생성 확인 | 자동 테스트 | 오류 없음 |

### 7.2 통합 테스트

| 시나리오 | 절차 | 합격 기준 |
|----------|------|----------|
| 전체 발표 흐름 | 로그인 → 업로드 → 발표 → 보고서 생성 | 오류 없이 완료 |
| 제스처 제어 | 슬라이드 전환 10회 | 오작동 ≤ 1회 |
| 분석 → AI 코칭 | 분석 후 Gemini 호출 | 코칭 정상 생성 |
| 보고서 생성 | PDF 생성 요청 | 10초 이내 완료 |

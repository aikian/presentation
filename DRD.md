## 1. Executive Summary

발표 능력은 학업과 취업, 직무 전반에서 요구되는 핵심 역량이다. 그러나 대부분의 발표자는 자신의 시선 처리, 자세 등 비언어적 습관을 객관적으로 인식하기 어렵고, 전문 코칭은 비용과 접근성 면에서 한계가 있다.

본 프로젝트는 노트북 내장 웹캠만으로 발표자의 시선, 자세, 제스처를 실시간 분석하고, 발표 종료 후 AI 기반 코칭 텍스트와 문제 순간 캡처 이미지가 포함된 PDF 보고서를 자동 생성하는 PC용 웹 애플리케이션이다.

핵심 차별점은 세 가지다.
첫째, 별도 하드웨어 없이 노트북 웹캠만으로 동작한다. 
둘째, 손 제스처로 슬라이드를 제어하면서 동시에 발표 분석이 이루어진다. 
셋째, 수치 점수에 그치지 않고 생성형 AI가 문제 발생 순간의 캡처 이미지와 함께 구체적인 행동 개선 코칭 문장을 생성한다.

주요 타겟은 발표 연습이 필요한 대학생, 취업 준비생, 직장인이며 별도 장비나 코치 없이 언제든 발표 역량을 객관적으로 진단하고 개선할 수 있다.

## 2. 개론

### 2.1 개발 배경 및 동기

발표 역량은 현대 사회에서 필수적인 능력으로 요구되고 있다. 그러나 발표자는 스스로 자신의 비언어적 습관(시선 회피, 자세 불안정 등)을 인지하기 어렵다. 기존의 전문 PT 코칭은 비용이 높고, 녹화 후 자기 분석은 객관성이 부족하다. 노트북이 있는 환경이라면 누구나 즉시 사용할 수 있는 발표 분석 도구의 필요성이 존재한다.

### 2.2 개발 목표

웹캠과 Google MediaPipe를 활용하여 발표자의 시선, 자세, 제스처를 실시간 분석하고, 생성형 AI 코칭을 결합한 통합 발표 피드백 시스템을 개발한다. 발표 중 문제 순간을 자동 캡처하여 이미지 근거와 함께 AI 코칭 텍스트를 제공하며, 발표 종료 후 PDF 보고서를 자동 생성하여 발표 역량을 객관적으로 진단하고 개선할 수 있도록 한다.

### 2.3 주요 기능 요약

- 회원가입 및 로그인, 발표 기록 히스토리 조회
- PPT/PDF 업로드 및 목표 발표 시간 설정
- 손 제스처 기반 슬라이드 제어 (주먹/손바닥) 및 레이저 포인터
- 실시간 시선·자세·제스처 분석 및 파생 지표 산출
- 발표 영상 전체 녹화 및 문제 순간 자동 캡처
- 생성형 AI 코칭 텍스트 생성 (캡처 이미지 + 텍스트 2단 레이아웃)
- 슬라이드별 시간 관리 점수 산출
- PDF 피드백 보고서 자동 생성

### 2.4 현실적 제한조건

- 웹캠 해상도 720p, 고정 앵글, 30fps, 깊이 정보 없음 → 상반신 중심 분석으로 제한
- 홍채 정밀 추적 및 전후 방향 흔들림 감지 제외 (웹캠 물리적 한계)
- 조명 환경에 따라 인식률 저하 가능 → 발표 전 웹캠·조명 사전 점검으로 보완
- 다중 사용자 동시 사용 미지원 (1인 1세션)
- 음성 분석은 7월 이후 고도화 항목으로 분류
- 녹화 영상은 보고서 생성 후 자동 삭제 (개인정보 보호)

### 3.3 경쟁 제품 분석

현재 발표 코칭 시장에는 Orai, Speeko, Yoodli 등 AI 기반 제품들이 존재한다. 그러나 이들은 모두 음성 분석 중심이라는 공통된 구조적 한계를 가진다.

Orai는 발화 명료도, 속도, 에너지, 필러워드를 분석하고 7일 무료체험 후 월 $12.99의 구독 모델로 운영된다. 다만 분석이 음성 트랙에 한정되어 있고, 실시간 영상 분석이나 슬라이드 연동 기능은 지원하지 않는다.

Speeko는 발화 속도, 필러워드, 에너지 수준에 대한 즉각적인 피드백을 제공하며, 사용자의 이전 연습 결과에 따라 학습 경로를 조정하는 적응형 커리큘럼이 특징이다. 그러나 iOS 전용 앱으로 제공되며, 마찬가지로 시선·자세 분석이나 슬라이드 제어 기능은 없다.

가장 유사한 포지셔닝을 가진 경쟁 제품은 Yoodli다. Yoodli는 웹캠을 통해 발표자의 시선, 제스처, 표정 등 비언어적 요소를 분석하고 실시간 피드백을 제공한다. Google Cloud Google for Startups 지원을 받은 상업 서비스로, 구글 미트·줌·팀즈 등 화상회의 플랫폼과의 통합에 강점이 있다. 그러나 발표 슬라이드를 직접 제어하는 기능이 없고, 웹캠 영상을 기반으로 시선과 자세를 분석하기는 하지만 문제 발생 순간의 프레임을 자동 캡처하여 AI 코칭 근거로 활용하는 기능은 존재하지 않는다.

정리하면, 현재 시장에 존재하는 제품들은 음성 분석 도구(Orai, Speeko)와 화상회의 코칭 도구(Yoodli, Poised)로 양분되어 있다. 발표 슬라이드 제어, 실시간 영상 분석, 문제 순간 자동 캡처, AI 코칭 보고서를 하나의 워크플로우로 통합한 제품은 현재 시장에 존재하지 않는다. 본 프로젝트는 이 공백을 타겟한다.

## 4. 설계 기능 (Function Tree)

```
웹캠 기반 실시간 발표 분석 프로그램
├── 1. 회원 관리
│   ├── 회원가입 (이메일/비밀번호)
│   ├── 로그인/로그아웃 (JWT)
│   └── 발표 기록 히스토리 조회
├── 2. 발표 환경 설정
│   ├── PPT/PDF 업로드 및 슬라이드 변환
│   ├── 목표 발표 시간 설정
│   └── 웹캠·조명 사전 점검
├── 3. 제스처 인식 및 제어 신호 생성
│   ├── 왼손 주먹 → 이전 슬라이드
│   ├── 오른손 주먹 → 다음 슬라이드
│   ├── 손바닥 → 레이저 포인터 모드

│   ├── 시선 분석
│   │   ├── 정면 응시율
│   │   ├── 시선 방향 편향 (좌/우)
│   │   └── 눈 깜빡임 빈도 → 긴장도 지수
│   ├── 자세 분석
│   │   ├── 어깨 기울기
│   │   ├── 상체 좌우 흔들림
│   │   ├── 상체 앞쏠림 각도
│   │   └── 손 위치 높이 → 제스처 적극성 지표
│   └── 제스처 분석
│       ├── 손 움직임 속도/가속도 → 에너지 지수
│       └── 양손 대칭성
├── 5. 영상 녹화 및 문제 순간 캡처
│   ├── 발표 전 구간 녹화 (MediaRecorder API)
│   ├── 분석값 임계치 초과 시 타임스탬프 자동 기록
│   ├── 발표 종료 후 해당 프레임 이미지 추출
│   └── 보고서 생성 후 원본 영상 자동 삭제
├── 6. 시간 관리
│   ├── 슬라이드별 발표 시간 기록
│   └── 목표 시간 대비 실제 시간 비교 및 점수 산출
├── 7. AI 코칭 텍스트 생성 (Gemini API)
│   ├── 분석 수치 + 캡처 이미지 → 프롬프트 구성
│   └── 캡처 이미지 옆 행동 개선 코칭 문장 생성
└── 8. PDF 보고서 자동 생성
    ├── 카테고리별 점수 및 레이더 차트
    ├── 구간별 분석 그래프
    └── AI 코칭 섹션 (캡처 이미지 + 코칭 텍스트 2단 레이아웃)
```

## 5. 해결 방안 비교

### 5.1 영상 분석 실행 환경

영상 분석을 서버에서 실행할 경우, 30fps 웹캠 스트림을 매 프레임 서버로 전송하면 네트워크 지연이 누적되어 실시간 분석이 사실상 불가능하다. Railway 같은 공유 서버 환경에서는 CPU 자원이 제한되어 있어 다수의 MediaPipe 모델(Hand + Face + Pose)을 동시 구동하기도 어렵다.

반면 브라우저 실행 방식은 이 문제를 근본적으로 해결한다. MediaPipe는 컴퓨터 비전 특화 라이브러리로, 얼굴 감지·손 추적·분할 등의 작업에서 전처리부터 추론·후처리까지 전체 파이프라인을 처리하며, 미드레인지 하드웨어에서도 안정적인 실시간 성능을 제공한다. 브라우저 내 추론은 서버로 데이터를 전송하고 응답을 기다릴 필요가 없어 지연 시간을 크게 줄이며, 민감한 데이터가 외부로 전송되지 않아 프라이버시 보호에도 유리하다. 특히 MediaPipe의 추론은 GPU에서 실행되므로 메인 스레드에 영향을 주지 않아 UI 응답성도 유지된다.

이러한 이유로 본 프로젝트는 MediaPipe를 브라우저(WASM + WebGPU)에서 실행하거나 서버에서 실행하는 방안 중 테스트 후 채택한다.

| 방안 | 설명 | 장점 | 단점 | 채택 |
|------|------|------|------|:----:|
| A. 서버 실행 | MediaPipe를 백엔드 서버에서 실행 | 클라이언트 부하 없음 | 네트워크 지연으로 실시간 처리 불안정, Railway CPU 제한으로 다중 모델 동시 구동 어려움 | 테스트 후 결정 |
| B. 브라우저 실행 (WASM) | MediaPipe를 브라우저에서 실행 | 지연 없는 온디바이스 추론, 서버 비용 0, 영상 데이터 외부 전송 없음, Google 공식 지원 | 클라이언트 성능 의존 (저사양 PC에서 프레임 드롭 가능) | 테스트 후 결정 |

### 5.2 AI 코칭 방식

규칙 기반 방식은 구현이 단순하지만, "시선이 왼쪽으로 치우쳤습니다"처럼 고정 메시지만 반복 출력하여 실질적인 행동 개선으로 이어지기 어렵다. 발표자가 어떤 상황에서 어떻게 바꿔야 하는지에 대한 구체적 맥락을 담을 수 없다.

Gemini API는 분석 수치와 캡처 이미지를 함께 프롬프트에 입력할 수 있어, 실제 문제 발생 순간의 시각적 근거를 기반으로 개인화된 코칭 문장을 생성한다. 예를 들어 "3번 슬라이드 발표 중 어깨가 오른쪽으로 기울어진 상태에서 30초 이상 지속됨"이라는 수치와 해당 프레임 이미지를 함께 제공하면, 단순 수치 피드백과는 다른 구체적이고 실행 가능한 코칭이 가능하다. 또한 Gemini API는 현재 무료 사용 범위(Gemini 1.5 Flash 기준 분당 15회 요청) 안에서 발표 종료 후 보고서 생성 시점에만 호출하면 되므로 추가 비용이 발생하지 않는다.

| 방안 | 설명 | 장점 | 단점 | 채택 |
|------|------|------|------|:----:|
| A. 규칙 기반 텍스트 | 조건별 고정 메시지 출력 | 구현 단순, API 의존성 없음 | 획일적 메시지, 이미지 기반 맥락 제공 불가 | |
| B. Gemini API | 분석 수치 + 캡처 이미지 → 코칭 문장 생성 | 문제 순간의 시각적 근거 기반 개인화 코칭, 무료 범위 내 운영 가능 | 발표 종료 시 API 호출 지연 발생 (수 초 내) | O |

### 5.3 음성 분석 방식

1학기 프로토타입 단계에서는 MediaPipe 세 모델(Hand + Face + Pose)을 브라우저에서 동시 구동하는 것 자체가 메모리와 CPU 점유율 측면에서 부담이 크다. 여기에 Whisper WASM까지 추가하면 일반 노트북에서 프레임 드롭이 발생할 가능성이 높고, 특히 한국어 인식 정확도도 서버 측 모델 대비 낮다. 이에 따라 음성 분석은 핵심 기능(시선·자세·제스처)이 안정화된 이후인 7월부터 서버 기반 Whisper로 착수하는 것으로 일정을 확정했다.

| 방안 | 설명 | 장점 | 단점 | 채택 |
|------|------|------|------|:----:|
| A. Whisper WASM (브라우저) | 브라우저에서 직접 실행 | 서버 자원 불필요 | MediaPipe 동시 실행 시 메모리 부족, 한국어 인식률 저하 | |
| B. Whisper 오픈소스 (서버) | Railway 서버에서 실행 | 한국어 인식률 안정적, 고품질 전사 가능 | 서버 자원 소모, 음성 데이터 서버 전송 필요 | (7월 이후 고도화) |

## 6. 개발 비용

### 6.1 인건비

SW 초급기술자 노임단가 기준 약 200,000원/일을 적용한다. 팀원 5명이 12개월간 주 20시간 환산 기준으로 참여하며 추정 인건비는 약 120,000,000원이다.

### 6.2 외부 서비스 비용

Google Gemini API, Google MediaPipe(오픈소스), Vercel, Railway, Supabase는 모두 무상 제공 범위 내에서 운영하며 추가 비용이 발생하지 않는다.

## 7. 시장성 분석

### 7.1 타겟 시장

주 타겟은 발표 연습이 필요한 대학생, 취업 준비생, 직장인이며 부 타겟은 스피치 학원 및 기업 사내 교육이다.

### 7.2 시장 현황 및 전망

국내 에듀테크 시장 규모는 2023년 약 10조원으로 연평균 15% 성장 중이다. 글로벌 프레젠테이션 소프트웨어 시장은 2024년 약 68억 달러 규모로 연평균 8% 성장이 예상된다. 전문 PT 코칭 서비스의 월 비용은 10~30만원 수준으로, 소프트웨어로 대체 시 높은 비용 절감 효과를 기대할 수 있다.

### 7.3 판매 방식 및 예측

웹앱 SaaS 구독 모델(월 9,900원)과 기업·학교 단체 라이선스를 병행한다.
